# 首页设计与实现计划

我们将构建一个极简、未来感且具有高度工程美学的首页。

## 1. 基础组件构建 (UI Components)
为了保持设计的一致性，我们将先创建基础 UI 组件：
- **`components/ui/button.tsx`**: 设计两种风格（Primary/Secondary），Primary 采用高对比度（黑底白字或白底黑字），Secondary 采用幽灵按钮或细边框风格。
- **`components/ui/container.tsx`**: 统一的页面宽度容器，确保在大屏上的阅读体验（参考 Apple/OpenAI 的最大宽度限制）。
- **`components/ui/section.tsx`**: 带有默认上下内边距的区块组件，确保呼吸感。

## 2. 核心模块开发 (Landing Sections)
我们将页面拆分为独立的模块组件，存放在 `components/landing/` 目录下：

### Hero 区域 (`Hero.tsx`)
- **视觉**: 巨大的留白，居中的核心标语。
- **内容**: "Building the Future of Digital Experience"（构建数字体验的未来）。
- **交互**: 文字渐显动画，跟随鼠标的微弱光效（可选）。

### 核心能力 (`Features.tsx`)
- **布局**: 模块化卡片网格 (Bento Grid 风格)。
- **内容**: 全栈工程、AI 集成、系统架构。
- **动效**: 滚动时卡片依次浮现。

### 理念与愿景 (`Philosophy.tsx`)
- **视觉**: 纯文字排版，类似杂志的阅读体验。
- **内容**: 强调"技术服务于人"。

### 技术栈展示 (`TechStack.tsx`)
- **视觉**: 极简的 Logo 墙或技术关键词列表，使用单色处理，避免杂乱的色彩。

### 底部 (`Footer.tsx`)
- **内容**: 简洁的 CTA 和版权信息。

## 3. 页面组装 (`app/page.tsx`)
- 将上述组件组合。
- 添加全局的滚动平滑效果（如果需要）。
- 确保响应式适配（Mobile First）。

## 4. 技术细节
- **动画库**: 使用 `framer-motion` 实现所有进入动画（FadeInUp）。
- **样式**: 使用 Tailwind CSS v4，配合 CSS Variables 定义语义化颜色（如 `--background`, `--foreground`）。
- **字体**: 使用已配置的 `geistSans`。
